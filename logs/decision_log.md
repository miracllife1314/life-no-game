## 初始決策（範例）

### 日期：2026-07-05
### 決策主題：
建立商用系統標準起手式模板的文件架構。

### 背景：
希望每個新專案一開始就具備完整 AI 協作規範，避免每天貼超長 Prompt、避免文件互相過期、避免高風險操作出事。

### 選項：
- A：每次開發都貼一大段規則 Prompt。
- B：建立 Router 制 + 單一事實來源 + 機器護欄，日常依任務讀文件。

### 最終決策：
採 B。

### 原因：
降低 token、降低注意力衰減、降低文件過期風險，並用機器護欄補強人工自律。

### 風險：
Skill 自動觸發非 100% 可靠；preflight 目前需手動執行（待 git init 後掛 hook）。

### 後續追蹤：
1. 專案 git init 後，將 preflight 掛成 pre-commit / CI。
2. 視情況演進到 Supabase CLI migration 流程。
