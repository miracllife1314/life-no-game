# 規劃書：多期報名 / 跨期角色（學員可同時參加兩期、小隊長可跨期）

> 目標：讓「同一個人」能在多個梯次同時存在——例如在 A 期當小隊長、B 期當學員，
> 或同時參加兩期的課程；並讓大隊長能在後台直接安排，不必靠邀請連結重新註冊。

---

## 1. 現況分析（為什麼現在不行）

### 資料模型（其實已支援多期）
- `profiles` 一筆 = 一次「**報名/參與**」（enrollment）。
- 同一個人用 `profile_id` 串起來；**多期 = 同一個 `profile_id` 有多筆 `profiles`、各自不同 `batch_id`**。
- 分數 `score`、寵物 `user_pets` 都是綁「**該筆報名**」（每期各自獨立）。

→ **結構面其實支援多期**，問題出在「**操作介面**」。

### 卡住的兩個點
| 問題 | 程式位置 | 說明 |
|---|---|---|
| 修行者下拉**依期數篩選** | AdminDashboard.tsx:2235 | 選 49 期就只列 49 期的人；49 期沒人報名 → 下拉空 → 無法指派 |
| 「分配」是**改同一筆報名** | page.tsx:1338 `update profiles` | 把人改到別期 = **搬走**，不是**新增一期**；原本那期會消失 |

### 結論
- 目前要讓人「多參加一期」，**只能靠邀請連結重新註冊**（那會 `insert` 一筆新報名）。
- 後台分配表單**只能編輯既有報名**，無法把現有的人「加入」到新梯次。

---

## 2. 優化目標

1. 大隊長能在後台「**把某個現有的人加入到某個梯次**」（產生新報名列）。
2. 支援：
   - 同一人**同時參加兩期**（兩筆 active 報名）。
   - 同一人**跨期不同角色**（A 期小隊長、B 期學員）。
3. 不破壞現有單期運作與計分。

---

## 3. 設計方案

### 核心：區分兩種操作
| 操作 | 行為 | 用途 |
|---|---|---|
| **編輯現有報名**（現有功能） | `update` 那一筆 | 改小隊、角色、狀態 |
| **新增報名到梯次**（要新增的功能） | `insert` 一筆新 `profiles`（**同 profile_id、新 batch_id**） | 把現有的人加入另一期 |

### 新功能 UI：「加入既有成員到梯次」
- 選「**人**」：下拉**跨梯次、用 `profile_id` 去重**列出所有人（顯示姓名＋已參加的期數），不要再被單一期數篩掉。
- 選目標**梯次 + 小隊 + 角色 + 狀態**。
- 按下 → 後端 `insert` 一筆新報名：
  - `profile_id` = 該人、`name`/`phone` 沿用、`batch_id` = 目標期、`team_id`/`role` = 選的、`score` = 0、新的 `user_pets` 自動建立。
- **防呆**：同一 `profile_id` 在同一 `batch_id` 已有報名 → 擋下（不可同期重複）。

### 計分 / 寵物（關鍵決策，需你拍板）
目前是「**每期各自獨立**」：每筆報名有自己的分數、自己的寵物、各自排行。
- **建議維持「每期獨立」**：因為每一期是**獨立的競賽**，分數/排行本來就該分開；同一人切換期數看各自的進度。（前台已有「期數切換」可切。）
- 若要改成「**跨期共用同一隻寵物/總經驗**」→ 那是大改（要把分數/寵物從綁「報名」改成綁「人 profile_id」），不建議，也通常不符合「分期競賽」的設計。

---

## 4. 需要你決定的事

1. **雙期學員的分數/寵物**：維持「每期各自獨立」（建議），還是要「跨期共用」？
2. **小隊長跨期**：A 期當小隊長、B 期當學員 → 用「兩筆報名、各自角色」實現（建議）。確認這符合你的需求。
3. **登入後切換**：同一人雙期，登入後用「期數切換器」切換兩期的面板——確認這個體驗 OK。

---

## 5. 工作項目（在測試區做）

1. 後端：新增 `handleEnrollExistingPerson(profileId, batchId, teamId, role, status)` → `insert` 新報名 + 防同期重複。
2. 前端：新增「加入既有成員到梯次」表單（人員下拉跨期去重）。
3. 修正修行者下拉：分配既有報名時維持篩選；新增報名時改為跨期列出。
4. 測試：雙期同時 active、跨期角色、切換器、各自計分、各自排行皆正確。

**預估**：約 1～2 天（含測試）。**風險中等**（動到報名/角色核心），務必**先在測試區做、課程空檔上線**。
