# 規劃書：大隊長後台「一鍵檢視學員帳號」（可返回）

> 目標：大隊長在後台點任一學員 → 直接看到「那位學員的個人面板」（神獸、經驗、任務狀態、
> 歷史、見證…）→ 一鍵「返回大隊長」。方便巡視每個帳號的內容。

---

## 1. 可行性分析（好消息：成本低）

- 大隊長登入後，`fetchData` **已經把所有 profiles / submissions / user_pets / score_logs 全部載入記憶體**。
- 所以「檢視某學員」= **拿那位學員的資料餵給現有的個人面板元件顯示**即可。
- **不需要重新登入、不需要重新抓資料、不需要密碼** → 切換瞬間完成、體驗順。

> 現有的「GM 模式角色模擬」只是切換「角色」(學員/隊長)，用的還是大隊長自己的資料；
> 這個新功能是「**切換成某一位具體學員的資料**」，是不同的東西，會獨立做、較乾淨。

---

## 2. 設計方案

### 核心：新增「檢視中」狀態 `viewAsUserId`
- 平常 `viewAsUserId = null`，後台正常運作。
- 點某學員的「檢視」→ 設 `viewAsUserId = 該學員 id`。
- 個人面板的資料來源從「大隊長自己」改成「被檢視的學員」：
  - `profile` = 該學員、`userPet` = 該學員的神獸、`submissions` = 該學員的打卡、分數/等級/4D 屬性都跟著該學員。

### 入口（兩個地方加「👁️ 檢視」按鈕）
- 「**學員名單管理**」每一列加一顆「👁️ 檢視帳號」。
- （可選）「**名冊概覽**」也加。

### 檢視中的畫面
- 最上方固定一條提示列：
  > 👁️ 你正在檢視【王小明】的帳號（唯讀） ── [ 返回大隊長 ]
- 下方就是該學員的「個人面板 / 排行 / 見證 / 課程」分頁，內容全是該學員視角。
- 按「返回大隊長」→ `viewAsUserId = null` → 回到後台。

### 唯讀保護（重要）
- 檢視中**所有操作按鈕停用**（打卡、提交證明、發見證貼文…一律不可按或不送出），
  避免大隊長不小心「幫學員打卡」造成資料錯亂。
- 純粹「看」，不「動」。

---

## 3. 需要你決定的事

1. **唯讀 vs 可代操作**：
   - **建議：唯讀**（只看不動，最安全）。
   - 若未來想要「幫學員補打卡」，後台本來就有「手動補登」功能，不需要靠這個檢視模式代操作。
2. **檢視範圍**：只看「個人面板」就好，還是連「排行/見證/課程」分頁也用該學員視角？
   - 建議：**全部分頁都用該學員視角**（最完整），反正資料都在。

---

## 4. 工作項目

1. 新增 `viewAsUserId` 狀態 + `viewedProfile` 計算（從已載入的 profiles 取）。
2. 個人面板等分頁的資料來源改為「`viewedProfile || currentUser`」。
3. 加頂部「檢視中」提示列 + 「返回大隊長」按鈕。
4. 學員名單/名冊每列加「👁️ 檢視」按鈕。
5. 檢視中停用所有寫入動作（唯讀）。
6. 測試：切換正確、返回正確、唯讀生效、各分頁資料正確。

**預估**：約 0.5～1 天。

---

## 5. 風險與建議時程

- **風險低**：純前端、唯讀、用已載入的資料，不動資料庫、不影響學員端。
- 因為風險低，**這項可以不必等測試區**，可在課程期間做（做完先用測試帳號驗證再用）。
- 唯一要小心的是「唯讀保護要確實」——確保檢視中不會誤觸發寫入。

---

## 6. 一句話總結

> 這是「**把已經載好的學員資料，換一份來顯示，並加一條返回列**」——
> 低風險、體驗順、很實用。建議做成**唯讀**版本。
