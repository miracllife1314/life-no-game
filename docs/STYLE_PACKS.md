# STYLE PACKS — 可選風格包與風格選擇機制

> 母版不綁死單一風格。本檔是風格庫的**單一事實來源**：
> `docs/BRAND_UI_SYSTEM.md` 管硬性品質規則（token、狀態、無障礙、防破版），
> 本檔管「這個專案要長什麼樣」。由 UI/UX 角色（`docs/roles/ui-ux-product-designer.md`）使用。

## 一、風格提問協定（何時問、怎麼問）

任務涉及**新頁面、新流程、首頁、會員中心、後台、報名、結帳、新手引導、品牌改版**，
而使用者尚未指定風格時，UI/UX 角色**不得直接自行猜測**，先以簡單選項詢問。

原則上只問這 5 題（保持簡單，不得一次丟出大量專業設計問題）：

```text
1. 想選哪一種風格？（附下方 A~E 選項與一句話說明）
2. 主要使用者是誰？
3. 這個頁面最重要的目標是什麼？
4. 希望給人什麼感覺？
5. 有沒有參考網站、品牌或圖片？
```

* 使用者也可以直接提供參考網站、品牌、圖片或想傳達的感覺——有參考物就以參考物為準。
* 專案已選定風格後記入 `docs/PROJECT_CONTEXT.md`（或專案 UI 章節），之後**不重問**，全站沿用。
* 後台工具型頁面若專案已有全站風格，直接沿用，不需每頁再問。

## 二、五個基本風格（A～E）

### A｜Premium Minimal｜高級極簡

* **視覺方向**：乾淨留白、精緻排版、低彩度、高價值感。
* **適合**：高單價、品牌、顧問與精品型產品。
* **禁忌**：資訊塞滿、彩色濫用、廉價漸層、過多裝飾。
* **AI 提示詞**：使用高級極簡風設計，大量留白、低彩度、精緻字距與排版，讓畫面有高價值感與品牌質感，避免任何廉價模板感。
* **起手 token**（換入 BRAND_UI token 區即可）：

```css
--color-primary: #18181b; --color-primary-hover: #000000; --color-primary-soft: #f4f4f5;
/* 搭配：陰影只用 --shadow-sm、留白加大（區塊間距至少 --space-12） */
```

### B｜Warm Professional｜溫暖專業

* **視覺方向**：親和、有溫度、可信任；柔和但不幼稚。
* **適合**：教育、親子、會員、學院與服務型產品。
* **禁忌**：過度可愛化、幼稚字體、低對比灰底灰字、雜亂配色。
* **AI 提示詞**：使用溫暖專業風設計，柔和色調、親和圓角與真實照片，畫面有溫度且可信任，但保持專業排版，不做成幼稚風。
* **起手 token**：

```css
--color-primary: #b45309; --color-primary-hover: #92400e; --color-primary-soft: #fffbeb;
/* 搭配：圓角以 --radius-lg 為主、頁面底可用暖白 #fdfbf7 */
```

### C｜Modern Business｜現代商務

* **視覺方向**：專業、俐落、資訊清楚，強調效率與可信度。
* **適合**：SaaS、後台與工具型產品。
* **禁忌**：過多動畫、資訊層級不清、危險操作沒有確認、表格難讀。
* **AI 提示詞**：使用現代商務風設計，重點是資訊清楚、操作有效率、狀態明確，列表支援搜尋篩選分頁，整體俐落可信。
* **起手 token**：即母版預設 token 值（`--color-primary: #2563eb` 系列，見 BRAND_UI §2），不需更動。

### D｜Conversion Focused｜高轉換行銷

* **視覺方向**：CTA 清楚、資訊節奏明確、分段式說服，強調行動與轉換。
* **適合**：報名、銷售、招生與活動頁。
* **禁忌**：CTA 太多互搶、賣點太散、段落太長、假見證假圖。
* **AI 提示詞**：使用高轉換行銷風設計，首屏清楚傳達價值主張與單一主 CTA，依「痛點→賣點→證明→行動」節奏分段，手機優先，降低行動阻力。
* **起手 token**：

```css
--color-primary: #c2410c; --color-primary-hover: #9a3412; --color-primary-soft: #fff7ed;
/* 搭配：primary 只給主 CTA 用（全頁唯一焦點色），次要按鈕走中性色；白字對比已達 WCAG AA */
```

### E｜Technology Premium｜高級科技

* **視覺方向**：現代、精準、具科技感。
* **適合**：AI、科技與創新產品。
* **禁忌**：**不得過暗、過度炫技或影響可讀性**；避免滿版粒子動畫與霓虹濫用。
* **AI 提示詞**：使用高級科技風設計，現代精準、適度科技感點綴，維持高可讀性與清楚操作，不過暗、不炫技。
* **起手 token**：

```css
--color-primary: #7c3aed; --color-primary-hover: #6d28d9; --color-primary-soft: #f5f3ff;
/* 搭配：可偏深色模式呈現，但文字對比仍須達 WCAG AA（BRAND_UI §11） */
```

## 三、擴充風格包（情境化，來自實戰）

### F｜高級學院風

* **適合**：課程平台、顧問品牌、個人 IP、教育系統。
* **視覺方向**：深色底、金色點綴、大留白、真實照片、清楚 CTA、信任感文案。
* **禁忌**：過度 emoji、廉價漸層、假圖、過亮金色、資訊塞太滿。
* **AI 提示詞**：使用高級學院風設計，深色底、金色點綴、大留白、真實照片優先，畫面要有信任感與課程品牌質感，不要做成廉價模板感。

### G｜遊戲任務風

* **適合**：打卡系統、排行榜、積分、寵物、成就、任務牆。
* **視覺方向**：卡牌、徽章、等級、進度條、任務狀態、輕動畫。
* **禁忌**：過度花俏、資訊難讀、遊戲感壓過操作、動畫太重。
* **AI 提示詞**：使用遊戲任務風設計，保留清楚操作與手機可讀性，加入卡牌、徽章、等級、進度條與任務感，但不要犧牲管理效率。

## 四、使用規則

1. 不論選哪個風格包，`docs/BRAND_UI_SYSTEM.md` 的硬性規則（design token、狀態完整、無障礙、§12 防破版）**一律適用**，風格不得凌駕品質門檻。
2. 風格落地方式：把該包的視覺方向轉成 token 值——A～E 各附「起手 token」可直接換入，之後只微調；擴充包（F/G）於實際使用時依同格式補值。不是散寫色碼。
3. 一個專案原則上**一個主風格**；前台行銷頁與後台工具頁可分別選（例：前台 D + 後台 C）。

