本規劃書已針對您升級至 **Supabase 付費版（Pro Tier / 付費 S）** 進行了優化調整。付費版為本系統帶來了更高的安全性、穩定性與效能保障，解鎖了許多原本受限的優化方案。

## 🚀 付費版 Supabase 帶來的變更與優勢

1.  **無痛簡訊驗證碼 (Phone OTP) 成為首選**：
    *   *以前（免費版）*：每天有 10 封簡訊的限制，極易爆表。
    *   *現在（付費版）*：簡訊發送限額大幅提升（每個月提供大額度免費額度，超出後費率極低），可以直接對接簡訊服務。**我們強烈建議採用簡訊 OTP 登入**，這是學員體驗最好、最不需要記密碼的登入方式。
2.  **資料庫不再進入休眠 (No Pausing)**：
    *   *以前（免費版）*：若系統一週無人登入，資料庫會自動暫停（Sleep）。第一個打開網頁的學員需要等待 30 秒至 1 分鐘讓資料庫喚醒，否則會超時白畫面。
    *   *現在（付費版）*：資料庫 24 小時不間斷在線，網頁開啟將極速且穩定。
3.  **自動每日備份啟用**：
    *   *以前（免費版）*：沒有備份，管理員手忙腳亂。
    *   *現在（付費版）*：Supabase 官方每日自動備份資料庫並保留 7 天，遇到突發狀況可一鍵還原，維運風險趨近於零。
4.  **相片儲存容量與流量大解放**：
    *   *以前（免費版）*：Storage 限制 1GB。
    *   *現在（付費版）*：Storage 起步 100GB、資料庫 8GB。學員可以自由上傳高清的打卡證明相片，不用再限制極小的圖片壓縮率，使用者體驗更佳。

## 🔒 v2 核心變更：從「模擬登入」到「真實 Auth 權限驗證」

### 1. 簡訊 OTP 登入流程設計
學員首次使用時，只需輸入手機號碼並接收 6 位數簡訊驗證碼，即可自動建立/登入帳號：
```
學員網頁 ➔ 輸入手機 ➔ 呼叫 supabase.auth.signInWithOtp() ➔ 簡訊發送驗證碼
  ➔ 輸入驗證碼 ➔ 呼叫 supabase.auth.verifyOtp() ➔ 取得 JWT Token 存入 Cookie
```

### 2. 帳號關聯對接 (Data Schema)
*   **auth.users (Supabase 內建表)**：存放登入帳號、UUID 與手機號碼。
*   **public.profiles (現有學員表)**：新增 `auth_user_id uuid references auth.users(id)` 欄位。
*   **權限角色表 `public.user_roles` (新表)**：
    ```sql
    create table public.user_roles (
      auth_user_id uuid primary key references auth.users(id) on delete cascade,
      role text not null default 'student' -- student | captain | admin
    );
    ```

### 3. 資料表權限（RLS 政策）收緊配置
付費版帶來的強大 CPU 效能可以完美支撐更細緻的 RLS 政策，我們將收緊以下規則：

| 資料表 | 讀取權限 (SELECT) | 寫入/更新權限 (INSERT / UPDATE) |
|---|---|---|
| `profiles` | 本人（`auth.uid() = auth_user_id`）或管理員可讀。 | 僅限本人更新個資（姓名/電話）；僅限管理員更新分數。 |
| `submissions` | 本人、同隊隊長、大隊長可讀。 | 僅本人可新增打卡；僅小隊長/大隊長可更新審核狀態。 |
| `score_logs` | 本人、小隊長、大隊長可讀。 | **前端完全禁止寫入**！僅限資料庫 Trigger 於任務審核通過後自動寫入。 |
| `witness_likes` | 所有人可讀。 | 僅限本人按讚或取消按讚。 |
