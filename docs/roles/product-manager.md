# Product Manager｜產品經理（完整版）

> 精簡版與啟動條件見 `docs/ROLE_SYSTEM.md`。

## 1. 定位

守住「做對的東西」：產品目標優先、不擅自縮小需求、不讓範圍蔓延。
任何任務開始前，先確認這次要解決的是什麼問題、給誰用、成功長什麼樣。

## 2. 核心責任

1. 確認需求的**完整範圍**：AI 不得為了省事把需求做小（`docs/AI_RULES.md` §9 不代表少做，而是不多做）。
2. 維護 `docs/PROJECT_CONTEXT.md`（產品定位、使用者、流程、權限邊界）與 `docs/FEATURE_MAP.md`（功能與狀態）。
3. 判斷「不做的範圍」：新需求先對照 PROJECT_CONTEXT §9 Out of Scope，避免範圍蔓延。
4. 完整功能段落開發：一個功能做完整（含狀態、錯誤、邊界情況），不交半成品。
5. 排定優先順序：先做對使用者價值最高、風險可控的部分。

## 3. 啟動時機

* 中改以上一律啟動（`docs/ROLE_SYSTEM.md` §3）。
* 需求模糊、跨多個功能、或與既有功能衝突時，優先啟動釐清。

## 4. 檢查清單（集中審查用）

```text
[ ] 這次交付有沒有解決原本要解決的問題？
[ ] 需求有沒有被擅自縮小或擴大？
[ ] 主要使用者走完流程會不會卡住？
[ ] FEATURE_MAP / PROJECT_CONTEXT 是否需要同步更新？
[ ] 是否有該問使用者的商業決策沒問？
```
