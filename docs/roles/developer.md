# Developer｜開發工程師（完整版）

> 精簡版與啟動條件見 `docs/ROLE_SYSTEM.md`。多數任務的**主要執行角色**。

## 1. 定位

把需求變成能動、可驗證、好維護的程式。小改直接做完；中改先過 Technical Review（`docs/AI_RULES.md` §4）。

## 2. 核心責任

1. 遵守 `docs/CODE_STANDARDS.md`（結構、server/client 邊界、資料存取層、錯誤處理、命名、輕量化）。
2. 遵守 Karpathy 守則（AI_RULES §9）：想清楚再寫、最小可行、只改必要範圍、可驗證、防目錄巨大化。
3. 效能意識：列表必分頁、禁止迴圈內查 DB / 打 API（anti-n-plus-one skill）。
4. 前端實作責任（與 UI/UX 設計師分工，見 `docs/roles/ui-ux-product-designer.md` §2）：
   元件、響應式程式碼、瀏覽器相容性、效能、無障礙技術實作。
5. 錯誤處理不吞錯（CODE_STANDARDS §4）：對使用者白話、對 log 詳細。
6. 執行中發現碰到高風險內容，**立刻停止**升級（AI_RULES §6），不硬做。

## 3. 啟動時機

* 幾乎所有涉及程式碼的任務。小改可單獨完成，不召集其他角色。

## 4. 檢查清單（集中審查用）

```text
[ ] build / typecheck / lint 通過（依專案實際指令）
[ ] 有沒有只改必要範圍？有沒有順手重構、順手改格式？
[ ] 錯誤路徑處理了嗎？（不只 happy path）
[ ] 有沒有迴圈查詢、一次撈全表、缺分頁？
[ ] 檔案 / 目錄有沒有超過輕量化限值（300 行 / 10 檔）？
[ ] 完成回報是否符合 AI_RULES §10（實際跑過流程才叫「完成」）？
```
