# NLP 修行系統：四維指標規則與優化建議

本文件詳細說明目前系統中「四維指標」的計算規則、現有機制的限制，並針對未來遊戲化（Gamification）升級提出優化方案與程式碼實作建議。

---

## 一、 目前的四維指標規則

在現行系統中，學員個人儀表板「定課打卡」分頁（[DailyQuestsTab.tsx](file:///Users/leo/Desktop/定課系統/NLP_GAME/components/Tabs/DailyQuestsTab.tsx#L356-L360)）展示了四種修為屬性：
1. 👁️ **感官敏銳度** (Sensory Acuity)
2. ⚓ **心錨穩定度** (Anchor Stability)
3. 🤝 **親和感共鳴** (Rapport Resonance)
4. 🧩 **信念重塑力** (Belief Reshaping)

### 📊 當前計算公式
目前這四個指標的成長**完全由學員的「總修行分數」 (Total Score) 線性決定**，並為每個指標設定了不同的「基礎值」與「成長除數」（上限均為 100%）：

```typescript
// 程式碼位置：components/Tabs/DailyQuestsTab.tsx L357-L360
const attrAcuity = Math.min(100, 30 + Math.floor(profile.score / 150));
const attrStability = Math.min(100, 25 + Math.floor(profile.score / 180));
const attrRapport = Math.min(100, 40 + Math.floor(profile.score / 120));
const attrReshaping = Math.min(100, 20 + Math.floor(profile.score / 200));
```

| 指標名稱 | 基礎值 | 成長速率 (每獲得X分增加1%) | 達到 100% 所需總分 |
| :--- | :---: | :---: | :---: |
| 👁️ **感官敏銳度** | 30% | + 1% / 150分 | **10,500 分** |
| ⚓ **心錨穩定度** | 25% | + 1% / 180分 | **13,500 分** |
| 🤝 **親和感共鳴** | 40% | + 1% / 120分 | **7,200 分** |
| 🧩 **信念重塑力** | 20% | + 1% / 200分 | **16,000 分** |

## 二、 目前設計的限制（為什麼需要優化？）

1. **同質化成長，缺乏個性化特色**：
   因為四個指標都只跟「總分數」掛鉤，不論學員是練習感官、練習心錨，還是做其他限時任務，這四個指標都是**集體按固定比例上升**。這無法反映出學員個人的修行偏好（例如：某學員特別專注於心錨練習）。
2. **缺乏遊戲策略性**：
   學員不需要去思考「我應該多做哪類任務來提升特定的屬性」，降低了定課系統的遊戲化樂趣。
3. **與任務內容脫節**：
   任務本身沒有定義其「屬性傾向」，使指標顯得只是單純的進度條裝飾。

## 三、 優化建議方案

為了提升系統的擬真度與遊戲化體驗，建議將四維指標與**任務類別 (Task Category)** 或**神獸進化 (Pet System)** 進行實質性的數據綁定。

### 💡 方案 A：基於「任務分類」的定向成長（最推薦 ⭐️⭐️⭐️）
* **核心概念**：
  學員完成特定分類的任務，只會成長該分類對應的四維屬性。
* **具體規則**：
  1. 在任務/任務模板中正式引入 `category` 欄位（例如：`acuity`、`stability`、`rapport`、`reshaping`）。
  2. 計算屬性時，統計學員**已完成且審核通過**的特定類別任務積分。
  3. **新公式範例**：
     - `感官敏銳度` = 基礎值 + `已核准的 acuity 任務總分` / 成長除數
* **玩家體驗**：
  學員如果多打卡「每日五感恩（感官敏銳類）」，其「感官敏銳度」就會明顯高於其他屬性，真正實現個性化的修行面板。

### 💡 方案 B：與「神獸進化路線」關聯加成（⭐️⭐️）
* **核心概念**：
  根據學員孵化出的神獸路線（龍、獅、狐、靈獸），對對應的屬性進行係數加成或解鎖特殊上限。
* **具體規則**：
  1. **影響力龍系**：對「感官敏銳度」與「信念重塑力」提供成長速度 +20% 的加成。
  2. **行動力獅系**：對「信念重塑力」與「心錨穩定度」提供額外成長。
  3. **親和力狐系**：對「親和感共鳴」提供額外成長（例如基礎值直接提升至 60%）。
  4. **穩定靈獸系**：對「心錨穩定度」與「親和感共鳴」提供成長加成。
* **玩家體驗**：
  神獸不再只是視覺寵物，而是切實影響玩家能力屬性的修行夥伴。

### 💡 方案 C：連續定課打卡加成與衰退機制（⭐️⭐️）
* **核心概念**：
  引入「連續打卡」的熟練度，若連續定課，屬性會獲得額外 Buffer；若中斷定課，屬性會稍微退化。
* **具體規則**：
  - 檢測過去 3 天內是否有打卡。若有，保持正常屬性；若連續 3 天未打卡，四維指標每天自動衰減 5%（但總經驗值不變），直到重新打卡為止。
* **玩家體驗**：
  激勵學員保持每日修行紀律。

## 四、 方案 A（分類成長）的實作參考代碼

如果決定實施 **方案 A (基於任務分類成長)**，以下是修改方向的草案：

### 1. 資料庫與任務定義
在後台建立任務模板時，為每個模板指派分類：
- `每日五感恩` -> 分類：`acuity` (感官敏銳)
- `自我心錨重設` -> 分類：`stability` (心錨穩定)
- `呼應與親和感建立` -> 分類：`rapport` (親和感)
- `限制性信念破除` -> 分類：`reshaping` (信念重塑)

### 2. 前端計算邏輯修改 ([DailyQuestsTab.tsx](file:///Users/leo/Desktop/定課系統/NLP_GAME/components/Tabs/DailyQuestsTab.tsx))
```typescript
// 1. 篩選出所有已核准的打卡紀錄，並提取其對應任務的類別與積分
const approvedSubmissions = submissions.filter(s => s.status === 'approved');

// 2. 計算各類別的累計得分
let scoreAcuity = 0;
let scoreStability = 0;
let scoreRapport = 0;
let scoreReshaping = 0;

approvedSubmissions.forEach(sub => {
  // 尋找任務或任務模板
  const task = tasks.find(t => t.id === sub.mission_id);
  const mission = !task ? missions.find(m => m.id === sub.mission_id) : null;
  const category = task?.category || mission?.category;
  const score = sub.score_awarded || task?.score || mission?.points || 0;

  if (category === 'acuity') scoreAcuity += score;
  else if (category === 'stability') scoreStability += score;
  else if (category === 'rapport') scoreRapport += score;
  else if (category === 'reshaping') scoreReshaping += score;
});

// 3. 套用新的四維屬性計算公式
const attrAcuity = Math.min(100, 30 + Math.floor(scoreAcuity / 50));     // 由於是單類分數，除數要設小一點 (例如 50)
const attrStability = Math.min(100, 25 + Math.floor(scoreStability / 50));
const attrRapport = Math.min(100, 40 + Math.floor(scoreRapport / 40));
const attrReshaping = Math.min(100, 20 + Math.floor(scoreReshaping / 55));
```
