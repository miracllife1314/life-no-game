## B. 🟡 體驗優化（讓它更順、減少客服）

**前台（學員）**
- 任務列表**沒有空狀態**：某分類今天沒任務時是一片空白，學員以為壞了。加「今日尚無此類任務」。
- 大量原生 `alert()` / `confirm()`（進化、刪除等）在手機上醜且像當機，建議統一改成 App 內的精美提示。
- 四維屬性成長極慢（約 3 萬分才滿），學員會問「為什麼一直 1%」。加說明或重新調比例。
- 排行榜長名字（4 字以上）在獎台可能溢出。
- 成就頁固定深色、不跟隨淺色模式。
- 登出沒有確認、頭部按鈕觸控area < 44px，手機易誤按。

**隊長後台**
- 審核區**看不到證明圖片**：隊長在「初審區」按通過卻看不到截圖（放大鏡按鈕只在另一處）。要把看圖加進審核卡片。
- **「初審通過」其實是「最終通過、直接加滿分」**，措辭會誤導隊長以為只是初步。要改名或做成真的兩段式。
- 審核區沒有**待審數量提示**、不能批次通過、不能篩選。
- 達成率把**隊長自己**也算進隊員分母，會稀釋數字（要不要算，自己決定）。

**大隊長後台**
- 邀請功能不完整：只顯示邀請碼純文字，**不能複製完整邀請連結、不能開關招募、不能改人數上限**。要補「複製邀請連結」按鈕。
- `alert()` 與 toast 混用、且建立課程/公告/成就時**沒檢查錯誤**就報成功。
- 大額調分（例如手滑打 50000）沒有確認。

## C. 🟢 建議補上的功能（跑課程會用到）

1. **「未打卡名單」總覽**（最重要）：每天要能一眼看出某梯次誰還沒定課打卡。目前只看得到待審清單。
2. **取消審核 / 重新審核**：通過或駁回後無法反悔，手滑就回不去。
3. **已產生的任務可單筆編輯/刪除**（改日期、分數、刪除）。目前只能改模板重產，但重產會去重、改不到已存在的任務。
4. **隊長：聯絡隊員管道**（顯示電話 / 留言），現在隊長想關心落後的人沒有管道。
5. **AI 隊務分析是假的**（固定抓第 4 個人當「需要關懷」）→ **移除或明確標示為示範**，否則隊長會照假資訊行動。
6. **分數異動稽核表**：管理員看不到全體加扣分歷史（資料有，但只給學員自己看）。

## D. 效能 / 維運 / 安全

- **fetchData 每次載入撈 25 張表，且每次操作後又全部重撈一次**（page.tsx）。30-50 人 × 30 天後，submissions / score_logs 會到數千列，免費版會明顯變慢。建議：查詢按 `batch_id` 縮小範圍、操作後改「局部更新」而非整包重抓。
- **登入時整頁重新整理**（已知，你選擇暫不改）：會多跑一次完整載入。
- **安全（RLS 全開）**：已有 `docs/v2-加帳號驗證規劃.md`，課程空檔再做。
- **上線前先 `npm run backup`** 一次。
