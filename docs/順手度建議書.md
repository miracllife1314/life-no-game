## A. 輸入框摩擦（你舉的例子：數字預設 0 要先刪）🟢 最有感

全站共 **16 個數字輸入框，只有 3 個**做了「點擊自動全選」。其餘 **13 個**都是「點進去要先把舊的 0/1 刪掉才能打」，很煩。

**受影響的欄位（後台）：**
- 手動調分金額（`adjustAmount`）
- 成就門檻分數（新增 `achValue` / 編輯 `editAchValue`）
- 任務分數（`taskScore`）
- 神獸線任務：分數 `editLineTaskPoints`、可完成次數 `editLineTaskMaxCompletions`、排序 `editLineSortOrder`
- 隊長後台：手動補登次數（1–100 的輸入框）
- 其餘建立任務模板/課程的數字欄位

**建議做法（一次全改，零風險）：**
1. 每個數字框加「**點擊即全選**」（`onFocus` 選取），直接打就蓋掉舊值。
2. 進階：允許**清空成空白**再輸入（現在清空會跳回 0）。
3. 數字框統一「**點兩下選取 / 手機點一下選取**」行為一致。

> 這是 CP 值最高的順手度改進，我可以一次套用到全部 13 個。

## B. 設定方便性 🟡

1. **邀請連結無法一鍵複製**：後台只顯示邀請碼純文字，要自己拼 `?invite=...&batch=...&team=...` 很容易錯。建議加「**複製邀請連結**」按鈕，直接複製完整網址。
2. **「套用模板」與「產生任務」兩條路徑、一條沒提示**：用「套用模板」會靜默產生任務（沒有成功/數量提示），用「產生任務」才有。建議統一給「成功 X 筆 / 跳過 X 筆」的回饋，避免不確定到底有沒有產生。
3. **編輯表單預填全面檢查**：課程「報名連結」編輯不見的 bug 已修；建議順手把所有「編輯」表單都確認**打開時有正確帶入舊值**（避免同類 bug）。
4. **提示風格統一**：目前 `alert()`（建立課程/公告/成就）與 App 內精美 toast（打卡、小隊職責）混用。建議全部統一成 toast，手機上比較不突兀、也不會卡住畫面。

## C. 流程順暢 / 效能 🔵

1. **操作後整包重抓**：每次審核、調分、打卡後都重新載入全部 25 張表。資料變多會變慢。建議改「**局部更新**」（只更動有變的部分）。
2. **審核區看不到證明圖 + 無待審數量**：隊長/管理員審核時建議能直接看截圖、並在分頁顯示「**待審 N 筆**」紅點。
3. **「未打卡名單」總覽**：每天最需要——一眼看出誰還沒定課打卡。
